# 数据结构课程设计

## 项目简介

本项目是数据结构课程设计的综合实践，包含三个核心子系统，分别应用栈、队列、图、顺序表等经典数据结构，实现不同场景下的业务逻辑模拟：

- 停车场管理系统（栈 + 队列）
- 校园导航系统（图 + Dijkstra 算法）
- 员工信息管理系统（顺序表 + 快速排序）

所有系统均基于 C++ 控制台实现，聚焦数据结构的实际应用与算法逻辑验证。

## 子系统详情

### 1. 停车场管理系统

#### 核心问题

模拟单出入口狭长停车场（容量 N=2）的管理流程，支持车辆到达 / 离开、费用结算，区分小型 / 大型车计费规则（1 元 / 分、2 元 / 分），便道等候不计费。

#### 核心数据结构 & 算法

- 停车场：**顺序栈**（模拟后进先出，车辆离开需临时挪车）
- 便道：**链式队列**（模拟先进先出排队）
- 避让区：**临时栈**（存放挪车时的车辆，离开后归位）
- 核心逻辑：车辆到达 / 离开的栈 / 队列操作、费用计算（停留时间 × 车型费率）

#### 核心功能

- 车辆到达：车场未满入栈，已满则便道排队
- 车辆离开：挪开阻挡车辆→结算费用→车辆归位→便道车辆补位
- 费用结算：按车型和停留时间自动计算

### 2. 校园导航系统

#### 核心问题

将校园地图抽象为带权无向图，实现任意两点间的最短路径查询、连通性判断，解决校园内路径规划问题。

#### 核心数据结构 & 算法

- 校园地图：**邻接矩阵**（存储图的顶点 / 边及权重）
- 最短路径：**Dijkstra 算法**（贪心策略求解单源最短路径）
- 路径回溯：**栈**（倒推路径并正向输出）

#### 核心功能

- 景点名称→顶点索引映射
- 最短路径计算与距离统计
- 路径可视化输出（起点→途经点→终点）
- 连通性判断（无路径时提示）

### 3. 员工信息管理系统

#### 核心问题

实现员工信息的增删改查（CRUD）及多字段排序，支持按 ID / 姓名 / 职务等条件查询、排序，底层基于顺序表存储数据。

#### 核心数据结构 & 算法

- 数据存储：**顺序表（SqList）**（动态数组管理员工信息）
- 排序算法：**快速排序**（按 ID / 姓名 / 职务高效排序）
- 业务层封装：面向对象设计（Employee/Manager 类）

#### 核心功能

- 增：添加新员工（顺序表尾插）
- 删：按 ID 删除员工
- 改：更新员工职务 / 电话 / 住址等信息
- 查：按 ID / 姓名 / 职务查询员工
- 排：快速排序实现多字段排序
- 显：格式化展示所有员工信息

## 开发环境

- 编程语言：C++
- 开发工具：Visual Studio
- 运行环境：Windows 10/11 控制台

## 运行说明

1. 编译各子系统的源码文件（确保头文件引用正确）
2. 运行可执行文件，按控制台提示输入指令 / 参数
3. 各系统交互逻辑：
   - 停车场系统：输入 `A [车牌] [时间] [车型]`（到达）/ `D [车牌] [时间]`（离开）
   - 校园导航系统：输入起点 / 终点名称，自动规划路径
   - 员工管理系统：菜单式交互，选择数字指令执行增删改查

## 核心亮点

1. 贴合实际场景：每个系统均模拟真实业务逻辑，体现数据结构的工程价值
2. 算法落地：Dijkstra、快速排序等经典算法完整实现，验证正确性
3. 模块化设计：数据结构与业务逻辑解耦，代码易读、易扩展
